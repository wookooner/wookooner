📘 Project Guideline
Personal Digital Trace Manager

(자기 데이터 자기 관리 도구)

1. 프로젝트 목적 (Purpose)

이 프로젝트의 목적은 사용자가 자신의 인터넷 접촉과 디지털 흔적을 스스로 인지하고 관리할 수 있도록 돕는 것이다.

본 프로젝트는:

감시 도구가 아니다

타인의 정보를 수집하지 않는다

사용자를 대신해 행동하지 않는다

오직 사용자 본인의 동의 하에 생성되는 디지털 흔적을 정리·가시화·관리하는 것을 목표로 한다.

2. 해결하려는 문제 (Problem Statement)

대부분의 사용자는:

어디에 가입했는지 기억하지 못하고

어떤 사이트에 흔적을 남겼는지 인지하지 못하며

어떤 활동이 위험한지 판단하지 못한다

그 결과:

불필요한 계정 방치

과거 콘텐츠로 인한 리스크

데이터 유출 시 대응 지연

프라이버시 통제 상실

본 프로젝트는 이 문제를 기술적으로 복잡하지 않은 방식으로 해결한다.

3. 핵심 철학 (Core Principles)
3.1 Local-First

모든 데이터는 기본적으로 사용자 로컬 환경에 저장된다

클라우드는 필수 요소가 아니다

3.2 Data Minimization

필요한 정보만 수집한다

콘텐츠 내용, 입력값, 민감 정보는 저장하지 않는다

3.3 User Control

사용자가 언제든:

수집 범위를 조정할 수 있고

데이터를 삭제/내보낼 수 있어야 한다

3.4 Explain, Don’t Decide

시스템은 설명하고 제안한다

최종 판단과 행동은 항상 사용자가 한다

4. 범위 정의 (Scope)
4.1 포함 범위 (In Scope)

사용자의 웹 접촉 기록 수집

접촉 유형 분류

위험도 가시화

자동 삭제 정책

관리 가이드 제공

4.2 제외 범위 (Out of Scope)

타인 정보 수집

계정 자동 삭제/자동 결제

로그인 정보, 폼 입력값 저장

실시간 감시/추적 기능

블록체인 기반 신원 관리 (향후 고려)

5. 핵심 분류 모델 (Core Classification Model)

사용자의 인터넷 접촉은 다음 4단계로 분류된다.

5.1 View-only

읽기/시청/탐색

계정·콘텐츠 생성 없음

예: 뉴스 읽기, 유튜브 시청, 상품 구경

5.2 Account-touch

계정 존재 또는 인증 흐름 접촉

예: 로그인, 회원가입 페이지, 비밀번호 재설정

5.3 User-generated

사용자가 외부에 콘텐츠를 남김

예: 글 작성, 댓글, 리뷰, 업로드

5.4 Transaction

금전·법적·고도의 개인정보와 결합

예: 결제, 송금, 금융/정부/의료 서비스

이 분류는 사실 판정이 아니라 관리 기준이다.

6. 위험도 판단 원칙 (Risk Assessment)

위험도는 다음 두 축으로 판단한다.

Trace Intensity: 4단계 분류 수준

Site Sensitivity: 사이트의 민감도(금융/정부/일반 등)

위험도는:

경고의 강도

보존 기간

사용자 확인 필요 여부

를 결정하는 데 사용된다.

7. 데이터 보존 정책 (Retention Policy)

기본 정책 (사용자 변경 가능):

View-only
→ 단기 보존, 자동 삭제 (예: 30~90일)

Account-touch
→ 중기 보존, 계정 관리 대상

User-generated
→ 장기 보존, 정기 점검 대상

Transaction
→ 장기 보존, 핵심 자산으로 관리

8. 사용자 상호작용 원칙 (User Interaction Rules)

시스템은 항상 조용히 자동 분류를 우선한다

아래 조건에서만 사용자에게 질문한다:

고위험(Transaction) + 확신도 낮음

사용자 관리 가치가 높은 이벤트

질문은:

짧고

선택지 기반이며

이후에는 규칙으로 학습된다

9. 기술 방향성 (Technical Direction)
9.1 초기 구현

Chrome 확장 프로그램 기반

도메인/시간/빈도 중심 수집

룰 기반 분류 v1

9.2 단계적 확장

로컬 앱 연동

정교한 규칙

설명 생성용 AI 보조

(장기) 증명/신뢰 레이어로 블록체인 고려

10. 수익화 원칙 (Monetization Principles)

위험 감지 자체는 무료

대응·자동화·편의성에 비용 부과

보안 신뢰를 훼손하는 유료 제한 금지

11. 성공 기준 (Success Criteria)

이 프로젝트는 다음 질문에 “예”라고 답할 수 있으면 성공이다.

사용자가 자신의 디지털 흔적을 처음으로 인식했는가?

위험한 활동을 한눈에 파악할 수 있는가?

불필요한 데이터가 자동으로 사라지는가?

“이거 없으면 불안하다”는 반응이 나오는가?

12. 프로젝트 원칙 선언 (Final Declaration)

이 프로젝트는
기술 과시를 위한 것이 아니라
사용자의 통제권을 회복하기 위한 도구이다.
